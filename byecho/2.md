### 📊 智能柜全链路AI决策中枢架构图（Mermaid代码）

您可以直接复制下方代码块中的所有内容，粘贴到 [Mermaid Live Editor](https://mermaid.live/) （一个免费的在线生成工具），或者支持Mermaid的绘图工具（如 Draw.io、飞书文档、Notion）中，即可一键生成高清流程图，并导出图片插入PPT。

```mermaid
graph TD
    %% 定义样式
    classDef aiHub fill:#2D5A9E,stroke:#fff,stroke-width:2px,color:#fff,font-weight:bold;
    classDef aiBrain fill:#E67E22,stroke:#fff,stroke-width:2px,color:#fff,font-weight:bold;
    classDef dept fill:#F0F4F8,stroke:#2D5A9E,stroke-width:1px,color:#333;
    classDef external fill:#E8F8F5,stroke:#17A589,stroke-width:1px,color:#333;
    classDef data flow fill:#F9E79F,stroke:#D4AC0D,stroke-width:1px,stroke-dasharray: 5 5;

    %% 客户与起点
    Customer((客户需求)) -->|非标定制/批量订单| Sales[1. 销售部]
    class Customer,Sales dept;

    %% 核心枢纽：执行轮与认知轮
    subgraph AI双轮驱动中枢
        direction TB
        Hub{AI项目总控智能体\n执行轮 - 神经系统} 
        Brain((数据分析智能体\n认知轮 - 企业小脑))
    end
    class Hub aiHub;
    class Brain aiBrain;

    Sales -->|录入自然语言需求| Hub

    %% 并行研发协同
    subgraph 研发与资源协同 (端到端并行流)
        direction LR
        Admin[4. 行政部\n会议协调/样机流转/资源排期]
        Struct[5. 结构设计部\n空间校验/3D图纸/材质]
        Hard[6. 硬件部\n主板/除湿模块/电路BOM]
        Soft[7. 软件部\n控制程序/IoT协议开发]
    end
    class Admin,Struct,Hard,Soft dept;

    %% 任务分发与协同
    Hub ==任务拆解与指令分发==> Struct
    Hub ==任务拆解与指令分发==> Hard
    Hub ==任务拆解与指令分发==> Soft
    Hub -.调度资源.-> Admin

    %% 研发阶段整合
    Struct & Hard & Soft ==>|系统自动防冲突校验与BOM合并| Hub

    %% 供应链与生产交付
    subgraph 生产与供应链阶段
        Supplier[[8. 第三方生产供应商\n打样/开模/量产]]
    end
    class Supplier external;

    Hub ==智能询价/风险评估/自动派单==> Supplier

    %% 售后与运维
    subgraph 运维与售后阶段
        Ops[7. 运维部门\n故障预警/售后派单]
        Device((交付的智能柜\nIoT实时回传))
    end
    class Ops dept;
    class Device external;

    Supplier -->|完成生产与交付| Device
    Device -.运行数据流.-> Ops

    %% 辅轮：数据闭环与自我进化
    Ops -.异常报错/耗时记录.-> Brain
    Supplier -.交期延误/次品率数据.-> Brain
    Struct & Hard -.图纸打回/成本超标数据.-> Brain

    Brain -.深度挖掘与反思归因.-> Brain
    Brain ==输出优化策略/更新知识库==> Hub
```
